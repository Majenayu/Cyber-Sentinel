🐛 BUGS & BROKEN FEATURES 1. Mobile "Best AI" chat is broken cyber-sentinel-mobile/app/(tabs)/chat.tsx — The mobile useSendMessage hook uses plain fetch, but the Best-AI endpoint returns SSE (Server-Sent Events). Mobile gets no response from the best-AI route. 2. JSON.parse crash in analyze route api-server/src/routes/analyze.ts line ~58 — JSON.parse(jsonStr) is called with no try-catch. AI models frequently return malformed JSON; this will crash the entire request handler. 3. updatedAt mapped to createdAt in Knowledge route api-server/src/routes/knowledge.ts line ~25 — updatedAt is mapped as e.createdAt, so the "last updated" timestamp is always wrong in the Knowledge Base UI. 4. handleScraped receives array but expects string cyber-sentinel/src/pages/Vault.tsx UrlIngestRow — suggestedTags is returned as an array from the scraper, but handleScraped treats it as a comma-separated string, so tags are never correctly parsed when scraping URLs. 5. React key conflict in AnalyzeModal results cyber-sentinel/src/pages/Vault.tsx line ~139 — List items are keyed by array index i rather than a stable ID, causing mis-renders when results arrive rapidly. 6. Mobile has no image upload / analysis cyber-sentinel-mobile/app/(tabs)/chat.tsx — The 📎 file upload + Gemini Vision analysis exists in the web app but is completely absent in mobile. 7. Mobile has no "Stop Generation" button cyber-sentinel-mobile/app/(tabs)/chat.tsx — Long AI responses can't be interrupted; web has a stop button but mobile doesn't. 🔒 SECURITY ISSUES 8. SSRF in scraper (redirect bypass) api-server/src/routes/scrape.ts lines ~172-180 — fetch is called with redirect: 'follow' before any IP validation runs. A URL that redirects to 169.254.169.254 (AWS metadata) or any internal IP bypasses the private-IP check entirely. 9. Regex injection in analyze route api-server/src/routes/analyze.ts line ~151 — AI-generated command strings are inserted into MongoDB $regex with only partial escaping. A crafted command string can inject arbitrary regex patterns into the DB query. 10. Zero authentication on all API routes api-server/src/routes/ — Every single endpoint (chat, knowledge, commands, analyze, scrape) has no auth middleware. The API is completely open to anyone who can reach the server. ⚠️ MISSING ERROR HANDLING 11. Silent failure on Save in Commands page cyber-sentinel/src/pages/Commands.tsx lines ~60-69 — The handleSave catch block is empty. If the server is unreachable, the save button stops spinning with zero feedback to the user. 12. Short AI responses silently discarded api-server/src/lib/multi-ai.ts getBestAnswer — The filter requires content.length > 20. Responses shorter than 20 chars (valid one-word answers) are thrown away with no user-visible explanation. 13. Port startup race condition api-server/src/index.ts line ~6 — If API_PORT and PORT are both absent, the server throws immediately on startup before the process environment is fully loaded in some deploy environments. 🔀 INCONSISTENCIES & DEAD CODE 14. Duplicate Groq rate-limit tracking api-server/src/lib/groq-ratelimit-cache.ts AND api-server/src/lib/ai-limits-cache.ts — Both files track Groq rate limits. multi-ai.ts updates both (lines ~26-27). One is redundant; they can diverge and show conflicting numbers. 15. Model names in Settings don't match what's actually called api-server/src/routes/usage.ts hardcodes display model names (e.g. llama-3.1-8b-instruct:free) but multi-ai.ts has a fallback chain of 3 different models per provider — so the model shown in Settings is often not the model actually used. 16. UrlIngestRow dead code path cyber-sentinel/src/pages/Vault.tsx — UrlIngestRow is defined and rendered, but its handleScraped parent callback has the bug in issue #4, making the entire URL-ingest-to-vault flow non-functional end-to-end. 17. OTHER_SECRET_1 used as undocumented Cloudflare account ID api-server/src/lib/multi-ai.ts line ~336 — Cloudflare account ID silently falls back to OTHER_SECRET_1. No documentation, no warning logged. If OTHER_SECRET_1 gets rotated for a different purpose, Cloudflare silently breaks. 📐 TYPE & API ERRORS 18. @ts-ignore hiding broken mobile import cyber-sentinel-mobile/app/(tabs)/chat.tsx line ~19 — @workspace/api-client-react is imported with @ts-ignore. This likely means the package path or type declaration is broken in the mobile workspace; all type safety for API calls in mobile is suppressed. 19. Missing await on database connection in groq.ts api-server/src/lib/groq.ts buildMessages — connectToDatabase() is called with await, but there's no guarantee the KB query runs after the connection resolves in all code paths when called from getChatResponse without connection pre-established. 🖼️ UX PROBLEMS 20. White flash on load (FOUC) cyber-sentinel/src/App.tsx line ~22 — Forces dark class in JS after render. If the browser defaults to light, there's a visible white flash before React hydrates and applies the dark class. 21. Analyze modal progress stays visible after done cyber-sentinel/src/pages/Vault.tsx AnalyzeModal — onDone() refreshes the list but the progress/spinner state stays showing until the user manually closes the modal, making it look like the operation is still running. 22. No error display when all AI providers fail in stream api-server/src/routes/chat.ts stream endpoint — When the multi-AI fallback also fails, the error is written as a text: SSE chunk starting with ⚠. The frontend Chat.tsx renders it inline as if it's AI response text, with no distinct error styling. ⚙️ ENV / CONFIG ISSUES 23. Gemini image analysis uses hard-coded model URL api-server/src/routes/chat.ts /chat/analyze-image — Now updated to gemini-2.0-flash, but if that model is deprecated it will silently 404 with no fallback, unlike the multi-AI provider which has a 3-model chain. 24. No GROQ_API_KEY_2 fallback in groq.ts for streaming api-server/src/lib/groq.ts streamChatResponse — The getGroqClient() function was updated to prefer key-1 then key-2, but if key-1 hits a rate limit mid-stream (after the connection is open), there's no mid-stream recovery — it will throw and trigger the multi-AI fallback which re-sends the full message. 25. Mobile API base URL hard-coded cyber-sentinel-mobile/ — The mobile app likely hard-codes an API base URL or uses a relative path that only works in certain environments; no .env-driven base URL is visible, which means mobile will break when deployed to a different host.
